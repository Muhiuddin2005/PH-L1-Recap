# ‚úÖ **37-2 ‚Äî Different Ways to Handle Events in React**

React events work similarly to JavaScript events, BUT with a few rules:

* Event names use **camelCase** ‚Üí `onClick`, `onChange`
* You pass a **function reference**, not a function call.

Let‚Äôs break this into **when to use brackets**, **when not to**, **when to use arrow functions**, and **different event-handling styles**.

---

## **1Ô∏è‚É£ When NOT to Use Brackets**

If you write:

```jsx
<button onClick={handleClick}>Click</button>
```

This means:

* You are **passing the function reference**
* React will call it **only when the event happens**

‚úî Correct
‚ùå Does NOT run immediately
‚úî Best for simple handlers

Example:

```jsx
function handleClick() {
  alert("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>
```

---

## **2Ô∏è‚É£ When to Use Brackets ( )**

When you need to pass:

* parameters
* dynamic values
* custom arguments

Then you MUST write:

```jsx
<button onClick={() => handleClick("Muhiuddin")}>Click</button>
```

Why?

Because writing:

```jsx
<button onClick={handleClick("Muhiuddin")}>Click</button>
```

means:

* It will **run immediately**
* React tries to set the return value as the handler ‚Üí ‚ùå WRONG

So we use arrow function to **delay execution**:

‚úî The arrow function runs ONLY when clicked
‚úî Then calls your function inside

Example:

```jsx
function handleClick(name) {
  alert("Hello " + name);
}

<button onClick={() => handleClick("Muhiuddin")}>Say Hi</button>
```

---

## **3Ô∏è‚É£ When You MUST Use Arrow Function**

Use arrow function when:

### ‚úî Passing arguments

```jsx
onClick={() => doSomething(id)}
```

### ‚úî Using event object with your own parameter

```jsx
onClick={(event) => handleClick(event, id)}
```

### ‚úî Updating state inside inline

```jsx
onClick={() => setCount(count + 1)}
```

---

## **4Ô∏è‚É£ Inline Event Handler (Simple Actions)**

Best for quick actions:

```jsx
<button onClick={() => console.log("Clicked!")}>Log</button>
```

But avoid using long logic inside inline.

---

## **5Ô∏è‚É£ External Event Handler (Best Practice)**

```jsx
function handleSubmit() {
  console.log("Submit");
}

<button onClick={handleSubmit}>Submit</button>
```

---

## üìå Real-life analogy:

Think of `onClick={handleClick}` like giving someone your **phone number**.

They can call you when they need.

But `onClick={handleClick()}` is like you **calling them immediately** instead of waiting.

# ‚úÖ **37-3 ‚Äî Simple Introduction to State Change in React**

## **What is State?**

State = a value that React WATCHES.
Whenever it changes ‚Üí UI updates automatically.

---

## **Default Values for Different Data Types**

```jsx
const [name, setName] = useState("");       // string
const [age, setAge] = useState(0);          // number
const [isActive, setIsActive] = useState(false); // boolean
const [skills, setSkills] = useState([]);   // array
const [user, setUser] = useState({});       // object
```

React state MUST have an initial value.

---

## üìå Real-life example:

Your phone‚Äôs battery icon auto-updates.

* State = battery percentage
* UI = battery icon
* setState = charging updates the percentage

React is the same.

---

## Simple UI Example:

```jsx
const [count, setCount] = useState(0);

<button onClick={() => setCount(count + 1)}>Increase</button>
<p>Count: {count}</p>
```

Whenever you click:

* State changes
* React re-renders UI

# ‚úÖ **37-4 ‚Äî Understand useState & Hooks (with Real-life Analogy)**

## **What is a Hook?**

Hook = a special function that gives React extra power.

`useState` ‚Üí makes variable reactive
`useEffect` ‚Üí runs side effects
`useRef` ‚Üí stores values without re-render
`useContext` ‚Üí shares global values

---

## **useState Real-life Example: Water bottle volume**

* State = ‚Äúhow much water is in the bottle‚Äù
* UI = label on the bottle showing volume
* When you drink ‚Üí water reduces ‚Üí label updates

React example:

```jsx
const [water, setWater] = useState(100);

<button onClick={() => setWater(water - 10)}>
  Drink 10ml
</button>
<p>Remaining: {water} ml</p>
```

---

## Important Rules:

‚úî You MUST use hooks **inside component**
‚úî You MUST call hooks at the **top level**
‚úî Hooks NEVER work inside loops or conditions

# ‚úÖ **37-7 ‚Äî (Recap) Load data using async/await in React (Client Component)**

For normal React projects (Vite, CRA, RUET assignments):

Use:

* `useState`
* `useEffect`
* `async-await`

Example:

```jsx
const [posts, setPosts] = useState([]);

useEffect(() => {
  async function loadData() {
    const res = await fetch("https://jsonplaceholder.typicode.com/posts");
    const data = await res.json();
    setPosts(data);
  }

  loadData(); // ‚úÖ You must call the function
}, []);
```

---

### Real-life example:

Ordering food online:

* You place the order ‚Üí `fetch()`
* Wait ‚Üí `await`
* Food delivered ‚Üí UI updates with new data ‚Üí `setPosts(data)`
