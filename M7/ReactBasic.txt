# âœ… **4. map() Syntax Rule in React (very important)**

You wrote:


### â“ **What is the syntax rule here?**

### â“ **Is the second bracket mandatory?**

Yes. Letâ€™s break the syntax:


There are **two important brackets**:

---

## ğŸ§© **Bracket 1 â†’ (user) or user**

This is the **function argument**.

Arrow function can be:

```js

```


Both are OK because only 1 parameter.

## ğŸ§© **Bracket 2 â†’ ( ... )**

This bracket is **mandatory** if you return JSX in one line.

Because React must understand that:

`<p>...</p>` is a **return value**, not code.

If you remove this:

```jsx

```

BUT if you return multiple lines:

âŒ Wrong:

```jsx

  <p>{user.name}</p>
  <span>Hello</span>
```

JS thinks the function ended before returning.

âœ” Correct:

```jsx

  <>
    <p>{user.name}</p>
    <span>Hello</span>
  </>
)
```
