# üìù **Detailed Explanation of Each API**

---

# ‚≠ê **1. Link**

### **Purpose:** client-side navigation (no page reload)

```jsx
<Link to="/dashboard">Dashboard</Link>
```

* Replaces `<a href>` with faster SPA navigation.
* Does NOT give you **active styling**.

---

# ‚≠ê **2. NavLink**

### **Purpose:** navigation link that knows when it is active

```jsx
<NavLink 
  to="/dashboard"
  className={({ isActive }) => isActive ? "active" : ""}
>
  Dashboard
</NavLink>
```

* Adds `active` logic automatically.
* Perfect for menus, sidebars, tabs.

---

# ‚≠ê **3. useNavigate**

### **Purpose:** programmatically navigate (inside events)

```jsx
const navigate = useNavigate();

function handleLoginSuccess() {
  navigate('/profile');
}
```

Used when:

* Redirecting after form submit
* Redirecting after login
* Navigating based on conditions

---

# ‚≠ê **4. useNavigation**

### **Purpose:** monitor router navigation state

```jsx
const navigation = useNavigation();

if (navigation.state === "loading") {
  return <p>Loading...</p>;
}
```

`navigation.state` can be:

| State          | Meaning                 |
| -------------- | ----------------------- |
| `"idle"`       | No navigation happening |
| `"submitting"` | Form is being submitted |
| `"loading"`    | Router is loading data  |

Useful for:

* Global route loaders
* Top progress bar
* Disable buttons during pending navigation
