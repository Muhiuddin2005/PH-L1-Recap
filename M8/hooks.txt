# ‚úÖ **PART 1 ‚Äî What is a React Hook? (Super Beginner Friendly)**

### **A React Hook is simply a special JavaScript function that lets you use React features inside your component.**

Examples:

* `useState()` ‚Üí create and manage state
* `useEffect()` ‚Üí run side effects (API calls, timers, etc.)
* `useRef()` ‚Üí store values without re-rendering
* `useContext()` ‚Üí access global state

Hooks always start with **"use"** and can only be used inside React components or other hooks.

---

# ‚úÖ **Why create your own hook (custom hook)?**

You create a **custom hook** when you want to reuse logic.

Example:

* You have 3 inputs: name, email, password
* All of them need **state** and **onChange**

Instead of writing:

```js
const [name, setName] = useState("");
const [email, setEmail] = useState("");
const [password, setPassword] = useState("");
```

You create **one reusable hook**:

```js
const [name, handleNameChange] = useInputField("");
```

This reduces duplicate code.

---

# ‚úÖ **Rules of creating a React Hook**

### ‚úî 1. Hook names MUST start with `"use"`

Correct:

* `useInputField`
* `useAuth`
* `useDarkMode`

Wrong:

* `inputField()`
* `myHook()`

---

### ‚úî 2. Hooks can only be called inside:

* React components
* Other hooks

‚ùå You cannot call hooks inside:

* if statements
* loops
* normal functions

---

### ‚úî 3. Hook logic must always run in same order

React relies on order of hooks.
So don‚Äôt wrap hooks in conditions.

# ‚úÖ **PART 2 ‚Äî Explain Your Custom Hook (with inline comments)**

### üìå `useInputField.js`

```jsx
import { useState } from "react";

// Custom hook to handle an input field value
// defaultValue = the initial value of the input (usually empty string)
const useInputField = (defaultValue) => {

    // Create a state variable for this input
    const [fieldValue, setFieldValue] = useState(defaultValue);

    // Function that updates state when the user types
    const handleFieldOnChange = (e) => {
        setFieldValue(e.target.value);   // e.target.value = typed text
    };

    // Return BOTH the value and the change handler
    // So any component can easily use them
    return [fieldValue, handleFieldOnChange];
};

export default useInputField;
```

# ‚úÖ **PART 3 ‚Äî Explain HookForm Using Inline Comments**

```jsx
import React from 'react';
import useInputField from '../../hooks/useInputField';

const HookForm = () => {

    // Using our custom hook three times:
    // Each call returns: [value, onChangeHandler]
    const [name, nameOnChange] = useInputField('');
    const [email, emailOnChange] = useInputField('');
    const [password, passwordOnChange] = useInputField('');

    // When the form is submitted
    const handleSubmit = (e) => {
        e.preventDefault();              // stop page refresh
        console.log('submit', name, email, password);  // print values
    };

    return (
        <div>
            <form onSubmit={handleSubmit}>

                {/* Controlled field using custom hook */}
                <input
                    type="text"
                    defaultValue={name}
                    onChange={nameOnChange}  // updates name state
                />
                <br />

                <input
                    type="email"
                    onChange={emailOnChange}  // updates email
                />
                <br />

                <input
                    type="password"
                    onChange={passwordOnChange} // updates password
                />
                <br />

                <input type="submit" value="Submit" />
            </form>
        </div>
    );
};

export default HookForm;
```

| Feature / Point                   | **React Hooks**                                                           | **Normal JavaScript Functions**                                        |
| --------------------------------- | ------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **1. Purpose**                    | Used to manage React features (state, side-effects, refs, context)        | Used for general logic or calculations                                 |
| **2. Naming Rule**                | Must start with **`use`** (useState, useEffect‚Ä¶)                          | Can be named anything                                                  |
| **3. React Features Access**      | **Can** use React state, lifecycle, refs, context                         | **Cannot** use React features                                          |
| **4. Where They Can Be Used**     | Only inside React components or other hooks                               | Can be used anywhere (components, non-components, loops, conditions)   |
| **5. Rules to Follow**            | Must obey **Rules of Hooks** (no loops, no conditions, call at top-level) | No rules; can be called anytime, anywhere                              |
| **6. Re-render Trigger**          | Hooks **cause re-renders** when state updates                             | Normal functions **do not trigger** re-renders                         |
| **7. Example Use Case**           | `useState`, `useEffect`, custom hooks like `useInputField()`              | Utility functions, math functions, helper functions                    |
| **8. Memory Behavior**            | State persists between re-renders                                         | Variables reset each time function runs                                |
| **9. Return Value**               | Can return state + functions                                              | Can return anything (number, string, object)                           |
| **10. Side Effects**              | `useEffect` handles side effects                                          | Normal functions can run side effects anytime                          |
| **11. Who Controls Execution**    | React controls when hook runs                                             | You control when a normal function runs                                |
| **12. Can store component data?** | Yes, through state (persistent)                                           | Only temporary values (not persistent)                                 |
| **13. Scope**                     | Hook scope is tied to component lifecycle                                 | Function scope is independent                                          |
| **14. Reusability**               | Custom hooks allow reusable React logic                                   | Normal functions allow reusable general logic                          |
| **15. Can use JSX?**              | No ‚Äî hooks only contain logic, not JSX                                    | Also no ‚Äî normal functions cannot return JSX unless used as components |
